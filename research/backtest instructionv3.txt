## Step 1 — Core Performance Metrics

Calculate for **3Y, 5Y, and 10Y** windows ending 2026-03-31.

For **each window**, optimize weights using mean-variance (maximize Sharpe) with constraints, then report metrics at those optimized weights. Also report metrics at the **fixed proposed weights** (the table above) without optimization.

This is important: v2 reported optimized-per-window weights (e.g. J&J was 25.9% in 3Y but 5.0% in 10Y). Do the same here. Report both:
- **Row A:** Metrics at window-specific optimized weights (optimizer chooses within constraints)
- **Row B:** Metrics at fixed v3 proposed weights (no optimization)

| Metric | Formula |
|--------|---------|
| Ann. Return | `(final_value / initial_value) ^ (252 / n_trading_days) - 1` |
| Ann. Volatility | `daily_log_returns.std() * sqrt(252)` |
| Sharpe Ratio | `Ann_Return / Ann_Volatility` (rf = 0) |
| Max Drawdown | `max peak-to-trough decline on cumulative return series` |
| Calmar Ratio | `Ann_Return / abs(Max_Drawdown)` |
| Total Return | `(final_value / initial_value) - 1` |

**Output table columns:** Window | Type (Optimized/Fixed) | Sharpe | Ann Return | Ann Vol | Max DD | Calmar | Total Return

**Include v2 baseline for comparison:** 3Y: Sharpe 2.684, AnnRet 43.92%, MaxDD -28.75%. 5Y: Sharpe 2.033, AnnRet 32.68%, MaxDD -22.74%. 10Y: Sharpe 1.851, AnnRet 30.08%, MaxDD -24.86%.

## Step 5 — Floor Position Audit (Leave-One-Out)

For each of the 8 positions:
1. Remove it from the universe entirely
2. Re-optimize remaining **7 positions** (constraints: 5% min, 40% max, gold cap 25%, sum=100%) across **3Y, 5Y, 10Y**
3. Record the optimized Sharpe for each window
4. Calculate **ΔSharpe** = (Sharpe without this asset) − (Sharpe with all 8 assets) for each window. Positive ΔSharpe means removing it *helped* (bad sign for the asset).
5. Calculate pairwise **correlation to gold** (daily returns, 10Y)
6. Calculate pairwise **correlation to full portfolio** (daily returns, 10Y)
7. Count **regime wins** from Step 3

**Pass criteria** (same as v2):
- **Sharpe Test:** ΔSharpe ≤ −0.03 in **at least one** window (removing it hurts). Note: v2 header says "all windows" but the actual verdicts used "at least one" — use "at least one" which is the operative standard.
- **Corr Test:** Correlation to gold < 0.75
- **Regime Test:** Wins ≥ 4/6
- **Role Test:** Removing it doesn't eliminate the last position in its role category (Hedge/Carry/Convexity/Cyclical)

**Verdict logic:** KEEP if ≥3/4 criteria pass. REVIEW if 2/4. REJECT if ≤1/4.

**Output:** One row per position. Columns: Ticker, Name, Role, Weight, ΔSharpe 3Y, ΔSharpe 5Y, ΔSharpe 10Y, Corr Gold, Corr Port, Regime Wins, Sharpe Test (PASS/FAIL), Corr Test, Regime Test, Role Test, Verdict.

**Also include baseline row:** "Baseline 10Y Sharpe (all 8 positions): X.XXX | Ann Return: XX.X% | MaxDD: -XX.X% | Calmar: X.XXX"

## Step 6 — Gold Stress Test

**Method:** Apply a **permanent level shock** to the gold price series. Multiply the entire gold daily price series by (1 − shock%). This reduces all gold returns proportionally. Do NOT shock silver — they are separate assets. Other assets unchanged.

**Important for v3:** Silver is now 10% of the portfolio and correlated ~0.78 to gold. Add a **Combined Precious Metals Stress** (shock both gold and silver simultaneously). Do both the gold-only stress (for v2 comparability) AND the combined stress (new for v3).

**Gold-only stress (comparable to v2):**

For each shock level (−10%, −20%, −30%, −40%, −50%) × each gold cap level (10%, 15%, 20%, 25%, 30%, 35%):
- Portfolio weights are the optimized weights for that cap level (from Step 4, 10Y window)
- Recalculate portfolio total return, Sharpe, Ann Return, Max DD
- Calculate delta vs unshocked baseline for that cap level
- Calculate "gold drag" in pp

**Combined precious metals stress (new for v3):**

Same shock levels but applied to BOTH gold and silver simultaneously. Only test at the 25% gold cap level (the one we actually use). This answers: "If the debasement thesis fails and both gold and silver correct hard, how bad is it?"

**Output:**
1. Gold-only: Total Return matrix (cap × shock) — same format as v2
2. Gold-only: Sharpe matrix (cap × shock)
3. Combined: Total Return and Sharpe at 25% gold cap with both metals shocked

## Constraints & Standards

**Optimization:**
- Algorithm: SLSQP (scipy.optimize.minimize)
- Random restarts: 80 minimum per optimization run
- Initial weight generation: uniform random on simplex, respecting bounds
- Position bounds: 5% minimum, 40% maximum per position
- Gold hard cap: 25% (overrides 40% max — gold can never exceed 25%)
- Silver: subject to standard 5%–40% bounds (no special cap)
- Sum of weights: exactly 100%

**Calculations:**
- Sharpe: annualized return / annualized volatility (risk-free = 0%)
- Annualized return: geometric (compound), not arithmetic
- Annualized volatility: daily log return std × √252
- Max drawdown: calculated on cumulative daily return series, not monthly
- All returns: price only (no dividends), consistent with v2
- Portfolio return: weighted sum of individual daily log returns

**Data handling:**
- When Vertiv data unavailable: exclude and normalize remaining weights. Document every instance.
- Missing data: forward-fill up to 5 days. If gap >5 days, flag and document.
- Weekend/holiday alignment: use inner join on trading dates across all assets

**Formatting:**
- Percentages: 1 decimal (e.g., 30.1%)
- Sharpe: 3 decimals (e.g., 1.851)
- Basis point deltas: "+X.Xpp" or "−X.Xpp"
- Currency: no decimals, thousands separator (e.g., 10,469,331 kr)
- Clearly label every table with what it shows, the date range, and any exclusions
